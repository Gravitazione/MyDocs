#### Conventions and Guidelines

- **.husky** -> โฟลเดอร์สำหรับการเก็บ husky เพื่อใช้งาน git hook เช่นการทำ pre-commit, pre-push
- **.storybook** -> โฟลเดอร์สำหรับการเก็บ config storybook เพื่อใช้งาน storybook
- **test** -> โฟลเดอร์สำหรับการเก็บ unit test, สำหรับการทำ test ครอบ functions ต่างๆ
- **assets** -> โฟลเดอร์สำหรับเก็บ assets เช่น Images, Fonts
  - **fonts** -> โฟลเดอร์สำหรับเก็บ fonts
  - **images** -> โฟลเดอร์สำหรับเก็บ images (ข้างในแยกโฟลเดอร์ในมุมมอง feature base)
  - **icons** -> โฟลเดอร์สำหรับเก็บ icons (ข้างในแยกโฟลเดอร์ในมุมมอง feature base)
  - กรณีมี asset อื่นๆที่นอกเหนือจากที่กล่าวมาให้สร้างโฟลเดอร์ใหม่ภายใน asset ได้เลยเช่นมี asset ที่เป็น video ก็สร้างโฟลเดอร์ชื่อ videos
- **components** -> โฟลเดอร์สำหรับเก็บ Global component( component ที่ใช้ในหลายๆที่) 
โดยมุมมองในการแยกโฟลเดอร์มองเป็น object base เช่น โฟลเดอร์ button สำหรับใส่ Global component 
ที่เกี่ยวกับ button ถ้ามีการทำ Global component อื่นๆ ก็นำมาเก็บไว้ในโฟล์เดอร์ของ components
- **constant** -> โฟลเดอร์สำหรับเก็บ ข้อมูลที่เป็นค่าคงที่ต่างๆ
  - **enums** -> โฟลเดอร์สำหรับเก็บไฟล์ enums ต่างๆที่ใช้ในโปรเจค
  - **i18n** -> โฟลเดอร์สำหรับเก็บ internationalization หรือการ support หลายภาษา
- **interfaces** -> โฟลเดอร์สำหรับเก็บส่วที่เป็นเหมือน Data Model
- **types** -> โฟลเดอร์สำหรับการเก็บ type ต่างๆที่เฉพาะเจาะจง ใน typescript แยกตาม feature ต่างๆ
- **pages** -> โฟลเดอร์สำหรับเก็บ UI Page มีการแยกโฟลเดอร์โดยมองเป็น feature base ภายใน folder แบ่งเป็น components (สำหรับใช้ใน feature)
- **router** -> โฟลเดอร์สำหรับเก็บ router ต่างๆโดยแบ่งเป็นโฟลเดอร์ตาม feature และมาเรียกรวมกันในไฟล์ index.tsx ที่อยู่ระดับนอกสุดในโฟลเดอร์ router
- **services** -> โฟลเดอร์สำหรับเก็บ services api ต่างๆ
  - **api** -> โฟลเดอร์สำหรับเก็บ api โดยแบ่งตาม feature
  - **index.ts** -> ไฟล์ในการทำ services กลางสำหรับการเรียกใช้งาน api ต่าง
- **stores** -> โฟลเดอร์สำหรับการเก็บ Store เพื่อทำ State management แยกกัน ตาม feature 
  - **components** -> โฟลเดอร์สำหรับเก็บ Stores ของแต่ละ Component
  - **pages** -> โฟลเดอร์สำหรับเก็บ Stores ของแต่ละ Page
- **stories** -> โฟลเดอร์สำหรับเก็บ Storybook เพื่อการทำ UI testing, UI Documentation
- **styles** -> โฟลเดอร์สำหรับการเก็บ styles css, scss ของ Components หรือ Pages
- **utils** -> โฟลเดอร์สำหรับเก็บ Reusable function
  - **helpers.ts** -> ไฟล์สำหรับการเก็บ helper functions
  - **hooks.ts** -> ไฟล์สำหรับการเก็บ React hooks ที่เป็น Global
  - **utils.ts** -> ไฟล์สำหรับการเก็บ utility functions
- **.env** -> ไฟล์สำหรับเก็บ environment variables ต่างๆ
- **.eslintignore** -> ไฟล์สำหรับ config สิ่งที่จะ ignore จาก eslint
- **.eslintrc.cjs** -> ไฟล์สำหรับ config rules eslint
- **.gitignore** -> ไฟล์สำหรับ config สิ่งที่จะ ignore จาก git
- **.prettierignore** -> ไฟล์สำหรับ config สิ่งที่จะ ignore จาก prettier
- **.prettierrc.cjs** -> ไฟล์สำหรับ config rules prettier
- **tailwind.config.cjs** -> ไฟล์สำหรับ config tailwindcss
